Constraints :
constraints are the rule assigned to the columns to validate the data
types of constraints
 Unique, not null, check, primary key, foreign Key , default

Unique : unique is a constraint which does not accept duplicate or repeated values
 
2. Not Null : it is constraint which does not accept null values
              it makes columns as mandatory

3. Check : Check is an extra validation assigned to the column with condition if condition is true the values are accepted else not accepted
           Syntax : Check(col_name)

4. Primary Key :  primary key is a constraint which identify the unique records from the table.
                  primary key is a combination of unique and not null
                  a table should have only one primary key
                  primary key is also known as key attribute

5. Foreign key : foreign key is a constraint which is used to stablish connection between multiple tables
                 foreign key is not a combination of unique and not null
                 if any column wants to be a foreign key it must be a primary key in its own table
                 foreign key present in child table but it actually belongs to parent table
                 a table can have multiple foreign key
                 foreign key is also known as referential integrity constraint
 

Difference between primary key and foreign key :

 
Primary Key	                                          Foreign key
It identify the unique records from the table	          it is used to stablish relation between multiple tables
a table should have only one primary key	          a table can have more than one foreign key
primary key also known as key attribute	                  foreign key is also known as referential integrity constraint
primary key is a combination of unique and not null	  foreign key is not a combination of unique and not null


Default : it is used to assign default value for the column
             Syntax = default 'value'


Difference between unique and primary key :
Unique	                                                           Primary
a table can have multiple unique key	                           a table should have only one primary key
it ensure that there is no duplicates value in the column	   it identify the unique records from the table
Unique will accept null values	                                   primary key doesn't accept both duplicate and null values
a unique column can not be a foreign key in another table	   a primary key column  can be a foreign key in another table

function name

5. substr(): it is used to extract a part of string from the given original string
